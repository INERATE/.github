## What we believe

LLMs don't fail because they can't code. They fail because nobody told them
what **done** means, what **good** looks like, and when to **stop**. Inerate
exists to close that gap: every loop needs a base case —

```
if (done) return;
```

## What we build

| Project | What it is |
|---------|------------|
| **[Atelier](https://github.com/INERATE/atelier)** | The craftsman's framework for AI-built software: loop engineering with base cases, enforced clean-code law, anti-AI-slop design law, automatic model routing, SQLite knowledge graph, self-learning overlays. One install. |

```
/plugin marketplace add INERATE/atelier
/plugin install atelier
/atelier:bootstrap
```

## How we engineer

- **Done-states, not vibes** — every task terminates against a written,
  checkable contract.
- **Laws, not suggestions** — 250 words per file enforced by hooks; one accent
  color enforced by review; no polling, anywhere, ever.
- **Token economy** — knowledge lives in queryable rows and graphs, not
  re-read documents.
- **Premium by default** — if it looks like a template, we delete it.
